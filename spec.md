# 開発者向け仕様書（仮）
 本システムを構成する各Webアプリケーションについて，今後プログラムを改良する開発者を対象として，内部構造や処理の流れ，ページ遷移，リソース設計について説明する．

 ## ポケモン図鑑システム
 ### データ構造
  ポケモン図鑑管理システムでは，ポケモンのデータをJavaScriptの配列として管理している．各要素はオブジェクトであり，図鑑No.，名前，タイプ1，タイプ2，特性といった情報をプロパティとして保持する．データはサーバプログラム内の変数として管理されており，データベースは使用していない．

  - ポケモンデータ管理システム
    - ポケモンデータ配列
      - ポケモンオブジェクト
        - 図鑑No.
        - 名前
        - タイプ1
        - タイプ2
        - 特性


  ### ページ遷移
   一覧画面では，登録されているポケモンの一覧が表示され，ポケモン名をクリックすることで詳細画面へ遷移する．一覧画面には，新規登録画面へのリンクがあり，登録画面からデータを送信すると，一覧画面へ戻る構成となっている．詳細画面からは編集画面，削除処理を行うことができる．データ管理処理におけるページ遷移と内部処理の流れを図に示す．このフローチャートは，一覧画面から詳細画面を経由し，編集と更新後に一覧画面に戻る一連の処理を表している．なお，新規登録処理については一覧画面から新規登録画面を経由して一覧画面に戻るため，文章でのみの説明とする．

   ```mermaid
   stateDiagram-v2
    [*] --> 一覧表示
    一覧表示 --> 削除: 削除リンク
    削除 --> 一覧表示
    一覧表示 --> 詳細表示: ポケモン名をクリック
    詳細表示 --> 編集画面: 編集リンク
    編集画面 --> 更新
    更新 --> 一覧表示
    一覧表示 --> 新規登録画面: 追加リンク
    新規登録画面 --> 登録
    登録 --> 一覧表示
   ```


   ### HTTPメソッドとリソース名
   本システムでは，/pokemonをリソース名として使用している．GETメソッドは，一覧画面と詳細画面に，POSTメソッドは新規登録に使用している．

   ### 追加・編集・削除の表示
  追加，編集，削除のいずれの操作を行なった場合でも，処理が完了した後は選手一覧画面を表示する．ポケモン図鑑管理システムと同様の設計を採用している．

   ### リソースごとの機能詳細
  選手リソースも，一覧表示，詳細表示，追加，編集，更新，削除といったCRUD操作を実装している．
